This example creates GATT server and then starts advertising, waiting to be connected to a GATT client.

It uses ESP32\textquotesingle{}s Bluetooth controller and Nim\+BLE stack based BLE host.

This example aims at understanding GATT database configuration, handling GATT reads and writes, handling subscribe events, understanding advertisement and SMP related Nim\+BLE APIs.

It also demonstrates security features of Nim\+BLE stack. SMP parameters like I/O capabilities of device, Bonding flag, MITM protection flag and Secure Connection only mode, Enabling Link Encryption etc., can be configured through menuconfig options.

For RPA feature (currently Host based privacy feature is supported), use API {\ttfamily ble\+\_\+hs\+\_\+pvcy\+\_\+rpa\+\_\+config} to enable/disable host based privacy, {\ttfamily own\+\_\+addr\+\_\+type} needs to be set to {\ttfamily BLE\+\_\+\+ADDR\+\_\+\+RANDOM} to use this feature. Please include {\ttfamily ble\+\_\+hs\+\_\+pvcy.\+h} while using this API. As {\ttfamily ble\+\_\+hs\+\_\+pvcy\+\_\+rpa\+\_\+config} configures host privacy and sets address in controller, it is necessary to call this API after host-\/controller are synced (e.\+g. in {\ttfamily bleprph\+\_\+on\+\_\+sync} callback).

To test this demo, any BLE scanner app can be used.
